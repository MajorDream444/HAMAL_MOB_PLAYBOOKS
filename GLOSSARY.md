# HAMAL Glossary

Status: Stage A foundation

## HAMAL

The sovereign operating system that orchestrates Hanzo, AMA Solutions, and Lux.

HAMAL coordinates when to use AI infrastructure, when agents should operate, and when sovereignty or provenance should be recorded.

## MAIM

Major AI Mindset.

The education, community, ABC framework, Command Room, and knowledge-session layer.

MAIM teaches people how to use the system.

## AMA

AMA Solutions.

The agent operating layer for agent roles, playbooks, workspaces, memory, routing, and operational intelligence.

AMA operates.

## Hanzo

The AI infrastructure layer.

Hanzo provides model routing, tools, inference, execution primitives, APIs, and future runtime capabilities.

This repo does not redefine or modify Hanzo source code.

## Lux

The sovereignty layer.

Lux owns identity, credentials, provenance, ownership, blockchain, tokenization, digital assets, and verification.

## Context

The truth layer that defines how decisions are made.

## Agent

A role-bound digital operator with context, boundaries, tools, playbooks, rubrics, memory, and handoff rules.

## Skill

A documented method for solving a specific task.

## Playbook

A reusable operating recipe that sequences skills, tools, context, quality rules, and outputs toward a business result.

## SOP

A step-by-step procedure for consistent execution.

## Rubric

A quality standard for judging output.

## Tool

An executable capability such as Airtable, Notion, Google Sheets, or a repo utility.

## API

A connection layer between tools and external systems.

## MCP

A standardized interface for AI tool usage.

## Plugin

A bundled system of skills, tools, and configuration.

## Workspace

The operating area where agents think, remember, receive tasks, produce outputs, and hand off through files.

## Agent Contract

A file-based handoff agreement between agents, including producer, consumer, fields, validation, owner, and version.
