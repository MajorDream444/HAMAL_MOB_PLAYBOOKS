# Model / Influencer Directory + Platform System

## Purpose

The Model / Influencer Directory + Platform System is the bigger play behind the Model Brand Pressure Test and Model Monetization System.

It creates a structured intelligence layer for high-performance personal brands — models, influencers, athletes, fitness creators, beauty creators, lifestyle creators, and public-facing talent — so HAMAL can identify who has attention, who has weak backend systems, who needs support, and where platform-level opportunity exists.

## Core Thesis

Models, influencers, athletes, and creators are high-performance personal brands.

They may have visibility, but many do not own the system behind their visibility.

The directory is not just a list.

It is a market map for:

- talent discovery
- system gap diagnosis
- outreach prioritization
- offer routing
- ethical management
- brand deal infrastructure
- media kit generation
- content system installation
- future Lux/IP ownership layers

## Core Line

```text
Followers are not ownership.
Visibility is not a system.
```

## Scoring System

### System Gap Score

Measures how much backend structure is missing.

0 = no visible gap
100 = severe gap

Score signals:

- no media kit
- no booking link
- no public contact
- no clear offer
- no owned audience
- inconsistent content direction
- visible brand activity but no CRM/portfolio system
- high follower count with weak monetization path

### Opportunity Score

Measures commercial potential.

0 = low priority
100 = high priority

Score signals:

- strong visual identity
- strong niche/category
- evidence of brand deals
- quality content
- clear audience
- location relevance
- professional behavior
- likely pain around inconsistent income
- fit for Model Brand Pressure Test
- fit for Model Monetization System

## Score Bands

### 0–25 — Monitor

Low priority or not enough signal.

### 26–50 — Nurture

Potential, but not urgent.

### 51–75 — Strong Candidate

Good fit for outreach and pressure test.

### 76–100 — Priority Target

High attention + weak backend + clear monetization potential.

## Ethical Data Rules

Only use publicly available professional information.

Do not collect:

- private personal addresses
- private phone numbers
- private emails not publicly shared for business
- sensitive personal data
- minors' data without guardian/legal structure
- private images or non-consensual content

Do not automate scraping until:

- platform terms are reviewed
- data policy is created
- opt-out process exists
- Guardian approves

## Consent Rules

Before generating, editing, storing, or using likeness-based media:

- get explicit consent
- define usage purpose
- define storage location
- define expiration/revocation process
- define commercial rights
- keep record of permission
