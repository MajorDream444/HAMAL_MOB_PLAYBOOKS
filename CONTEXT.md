# HAMAL MOB Playbooks Context

Status: Stage A foundation
Owner: Major Dream Williams

HAMAL MOB Playbooks is the operating doctrine repo for context, playbooks, SOPs, rubrics, audits, and agent workspace architecture.

This repo does not run autonomous execution yet.

## Canonical Boundary

```txt
HAMAL orchestrates.
AMA operates.
Hanzo provides AI infrastructure.
Lux provides sovereignty.
MAIM teaches people how to use the system.
```

## What This Repo Owns

- agent context rules
- playbook doctrine
- SOPs
- rubrics
- tool and MCP audits
- industry systems
- MOB operating models
- agent workspace architecture
- reusable context templates

## What This Repo Does Not Own

- MAIM landing page code
- MAIM registration engine code
- Hanzo source code
- Lux contracts or blockchain implementation
- live automation execution
- production integrations

## Stage A Rule

Stage A is context and system foundation only.

No autonomous execution.

No live tool calls.

No secrets.

No production integrations.

## Operating Principle

```txt
No execution without context.
```

Every new tool, skill, repo, MCP, or agent must answer:

- What playbook does it serve?
- What context does it need?
- What agent or MOB owns it?
- What output does it create?
- Where does that output live?
- How is quality judged?
