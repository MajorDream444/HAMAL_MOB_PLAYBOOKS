# HAMAL Ecosystem Boundary

Status: Stage A foundation
Owner: Major Dream Williams

This document defines the boundary between HAMAL, MAIM, AMA, Hanzo, and Lux.

## Canonical Boundary

```txt
HAMAL orchestrates.
AMA operates.
Hanzo provides AI infrastructure.
Lux provides sovereignty.
MAIM teaches people how to use the system.
```

## HAMAL

HAMAL is the sovereign operating system.

It coordinates Hanzo, AMA Solutions, and Lux into usable operating systems, playbooks, and community intelligence.

HAMAL owns:

- orchestration
- doctrine
- playbook systems
- context architecture
- agent workspace architecture
- legacy intelligence
- cross-system coordination

## MAIM

MAIM is the education and community layer.

MAIM owns:

- ABC framework
- Command Room
- knowledge sessions
- community learning
- beginner AI education
- human-facing teaching assets

## AMA

AMA Solutions is the agent operating layer.

AMA owns:

- 144-agent architecture
- agent workspaces
- agent contracts
- playbooks
- SOPs
- rubrics
- routing
- handoffs
- operational memory

## Hanzo

Hanzo is AI infrastructure.

Hanzo provides:

- model routing
- AI services
- tool execution
- APIs
- inference
- infrastructure primitives
- future runtime capabilities

HAMAL can use Hanzo as a preferred AI infrastructure provider.

This repo does not redefine Hanzo and does not change Hanzo source code.

## Lux

Lux is the sovereignty layer.

Lux owns:

- identity
- credentials
- blockchain
- provenance
- ownership
- tokenization
- digital assets
- legacy verification

## Language Rules

Use:

```txt
HAMAL uses Hanzo as AI infrastructure.
AMA agents operate the workflow.
Lux provides sovereignty.
MAIM teaches the human layer.
```

Avoid:

```txt
Hanzo owns HAMAL.
Hanzo owns AMA workflows.
Hanzo is the whole operating system.
HAMAL changes Hanzo source code.
```
