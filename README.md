# HAMAL MOB PLAYBOOKS

HAMAL MOB PLAYBOOKS is the doctrine, systems, SOP, and playbook library for the HAMAL operating system.

This repository defines how agents understand context, use skills, call tools, follow playbooks, and serve specific industries.

## Purpose

This repo exists to turn raw knowledge into repeatable systems.

It organizes:

- context files
- SOPs
- playbooks
- skill templates
- rubrics
- MOB operating models
- industry systems
- repo / tool / MCP audits

## Core Belief

Skills alone do not create leverage.

Tools alone do not create leverage.

Agents become valuable when they have:

1. context
2. playbooks
3. tools
4. quality rules
5. feedback loops

## Stage

Current stage:

**Stage A — Context and System Foundation**

No autonomous execution should happen until the context layer, playbooks, SOPs, and audit structure are clear.

Canonical boundary:

```txt
HAMAL orchestrates.
AMA operates.
Hanzo provides AI infrastructure.
Lux provides sovereignty.
MAIM teaches people how to use the system.
```

## Repository Sections

This is the canonical HAMAL repo spine.

- `00_stage_a_foundation/` — foundation doctrine
- `01_sops/` — standard operating procedures
- `02_playbooks/` — reusable business playbooks
- `03_mobs/` — MOB-specific operating libraries
- `04_industries/` — industry-specific systems
- `05_templates/` — templates for context, playbooks, skills, SOPs, rubrics
- `06_audits/` — GitHub, skill, MCP, tool, and Hanzo/Lux audits
- `07_rubrics/` — quality standards
- `08_agent_context/` — context rules for agents
- `09_sources/` — notes from transcripts, videos, docs, and external sources

## Rule Before Adding Anything

Before adding a new tool, skill, repo, MCP, or agent, answer:

- What playbook does it serve?
- What context does it need?
- What agent owns it?
- What output does it create?
- Where does that output live?
- How is quality judged?
